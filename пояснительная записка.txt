# Кристальные Пещеры — Пояснительная записка

## 1. Описание проекта

**«Кристальные Пещеры»** — 2D-платформер на Python с использованием библиотеки **arcade 3.0**. Игрок управляет шахтёром, исследующим магические пещеры. Цель — собрать кристаллы, преодолеть препятствия, победить врагов и добраться до выхода через портал.

| Параметр | Значение |
|----------|----------|
| Жанр | 2D-платформер |
| Язык | Python 3.9+ |
| Библиотека | arcade 3.0 |
| Объём кода | ~1163 строки (один файл `main.py`) |
| Уровни | 3 |

---

## 2. Структура проекта

Весь код игры находится в одном файле `main.py`, разделённом на логические секции:

```
Arcade/
├── main.py              — весь код игры (единственный файл)
├── requirements.txt     — зависимости (arcade==3.0.0)
├── assets/sounds/       — звуки (генерируются автоматически при первом запуске)
└── high_scores.csv      — таблица рекордов (создаётся автоматически)
```

### Секции main.py

| Секция | Описание |
|--------|----------|
| **Константы** | Размеры экрана, физика, цвета, очки, пути к файлам |
| **Хранение данных** | Функции `save_score()` и `load_high_scores()` для CSV |
| **Генератор звуков** | Создание WAV-файлов программно через модуль `wave` |
| **Система частиц** | Классы `Particle` и `ParticleSystem` для визуальных эффектов |
| **Игрок** | Класс `Player` с анимацией ходьбы, прыжка, неуязвимости |
| **Враги** | Класс `Slime` — слизень с патрулированием и анимацией |
| **Построитель уровней** | Функция `build_level()` — 3 уровня из символьных карт |
| **Стартовый экран** | Класс `StartView` — заголовок, анимация, таблица рекордов |
| **Игровой экран** | Класс `GameView` — основной геймплей, камера, физика, коллизии |
| **Финальный экран** | Класс `GameOverView` — результаты, таблица рекордов |

## 3. Управление

| Клавиша | Действие |
|---------|----------|
| ← → или A/D | Движение влево/вправо |
| ПРОБЕЛ / ↑ / W | Прыжок |
| ESC | Меню / выход |
| ENTER | Начать игру / перезапуск |

## 4. Подсчёт очков

| Действие | Очки |
|----------|------|
| Жёлтый кристалл (C) | 100 |
| Голубой кристалл (B) | 200 |
| Зелёный кристалл (G) | 200 |
| Красный кристалл (R) | 500 |
| Уничтожение врага (прыжок сверху) | 250 |
| Завершение уровня | 500 |

**Здоровье:** 3 жизни. Потеря 1 жизни при столкновении с врагом сбоку. Контакт с лавой — мгновенная гибель. Сердца на уровнях восстанавливают здоровье.

## 5. Реализованные технологии

### 5.1. requirements.txt
Файл `requirements.txt` содержит единственную зависимость: `arcade==3.0.0`.

### 5.2. Стартовое окно (StartView)
Экран с названием игры, анимированными плавающими кристаллами, подсказкой по управлению и таблицей рекордов. Заголовок пульсирует, подсказка мерцает.

### 5.3. Финальное окно (GameOverView)
Экран завершения с двумя вариантами: победа (все уровни пройдены) и поражение. Показывает набранные очки, достигнутый уровень, таблицу рекордов. Текущий результат выделен.

### 5.4. Подсчёт и вывод результатов
HUD-панель в верхней части экрана: очки, текущий уровень, оставшиеся жизни (отрисованы как сердца). Всплывающие уведомления при сборе предметов и получении урона.

### 5.5. Спрайты
Все спрайты генерируются **программно** через PIL (Pillow): игрок-шахтёр (с каской и фонариком), слизни-враги, текстурированные стены, кристаллы (4 цвета), порталы, сердца, лава. Никакие внешние изображения не требуются.

### 5.6. Коллизии (collide)
Используется `arcade.check_for_collision_with_list()` для обнаружения столкновений игрока с:
- Кристаллами → сбор + начисление очков
- Врагами → урон сбоку ИЛИ уничтожение прыжком сверху
- Сердцами → восстановление здоровья
- Порталами → переход на следующий уровень
- Лавой → мгновенная гибель

### 5.7. Анимация
- **Игрок:** 4 кадра на каждое направление (покой, 2 кадра ходьбы, прыжок), мерцание при неуязвимости
- **Враги:** 2 кадра (сжатие/растяжение тела)
- **Интерфейс:** пульсация заголовков, мерцание подсказок, анимированные частицы на фоне
- **Переходы:** плавное затемнение между уровнями

### 5.8. Несколько уровней
Три уровня возрастающей сложности, заданные символьными картами:
1. **Обучение** — простой уровень без лавы, 2 врага
2. **Опасные пещеры** — появляется лава, 3 врага, сердце
3. **Финальное испытание** — максимальная сложность, 4 врага, лава, редкие кристаллы

### 5.9. Камера
`arcade.camera.Camera2D` — следит за игроком с ограничениями по краям. Отдельная GUI-камера для HUD, чтобы интерфейс не двигался при прокрутке уровня.

### 5.10. Система частиц
Собственная реализация на основе `arcade.SpriteCircle`:
- **Сбор кристалла** — разноцветный взрыв из 15 частиц с затуханием
- **Уничтожение врага** — облако пыли из 20 частиц
- **Контакт с лавой** — огненные искры

Каждая частица имеет случайное направление, скорость, время жизни и плавное затухание прозрачности.

### 5.11. Звук
8 звуковых эффектов, генерируемых программно модулем `wave` (синусоидальные тоны, свипы, шум):
прыжок, сбор кристалла, получение урона, уничтожение врага, портал, конец игры, лечение, лава.

### 5.12. Физический движок
`arcade.PhysicsEnginePlatformer` с гравитацией — обеспечивает:
- Падение игрока под действием гравитации
- Столкновение с платформами (стенами)
- Определение возможности прыжка (`can_jump()`)

### 5.13. Хранение данных (CSV)
Модуль CSV стандартной библиотеки Python. Файл `high_scores.csv` содержит поля: Имя, Очки, Уровень. Результаты сохраняются автоматически при завершении игры, загружаются и сортируются по убыванию очков для отображения таблицы рекордов.

## 7. Диаграмма классов

```
StartView (arcade.View)
  └── on_key_press(ENTER) → GameView

GameView (arcade.View)
  ├── Player (arcade.Sprite) — анимация, здоровье, неуязвимость
  ├── Slime (arcade.Sprite) — патрулирование, анимация
  ├── ParticleSystem — управление эффектами частиц
  │   └── Particle (arcade.SpriteCircle)
  ├── Camera2D — слежение за игроком
  ├── PhysicsEnginePlatformer — гравитация, столкновения
  └── on_transition → GameOverView

GameOverView (arcade.View)
  ├── save_score() → high_scores.csv
  ├── load_high_scores() ← high_scores.csv
  └── on_key_press(ENTER) → StartView
```

## 8. Заключение

Проект демонстрирует комплексное использование Python Arcade 3.0: физический движок, система частиц, камера, спрайтовая анимация, обработка коллизий, звуковые эффекты, многоуровневая архитектура и хранение данных в CSV. Все графические и звуковые ресурсы генерируются программно, что делает проект полностью автономным — для запуска достаточно установить `arcade` и выполнить `python main.py`.
